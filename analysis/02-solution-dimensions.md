# OhMyInfo — 解决方案维度分析

> 从 7 个关键维度对比不同解决方案的优劣
> 为 OhMyInfo 的产品定位和功能选型提供决策依据

---

## 维度 1: 信息采集策略

| 策略 | 描述 | 优点 | 缺点 | 代表项目 |
|------|------|------|------|---------|
| **RSS/Atom 订阅** | 标准 Feed 协议 | 稳定、标准化、无需 API Key | 部分网站无 RSS | Horizon, Signal, Feed Curator |
| **固定 API 调用** | GitHub/HN/Reddit/arXiv 官方 API | 结构化数据、可靠 | 有速率限制 | TechSentry, Sipply |
| **HTML 抓取** | 页面解析 (Cheerio/Scrapy) | 覆盖无 API 源 | 脆弱、易被墙 | Sentinel Feed, Keep Up Daily |
| **社交媒体** | Twitter/X API, 微信公众号 | 时效性强 | API 昂贵/受限 | Intel Briefing, FounderKit |
| **搜索增强** | Tavily/Exa/Google 搜索 | 覆盖广、发现新内容 | 成本高 | Signex, AI Newsletter Agent |

### 推荐策略 (OhMyInfo)

```
首选:  RSS + 公开 API (稳定、免费)
次选:  HTML 抓取 (补充覆盖)
按需:  LLM 搜索增强 (发现未知)
暂缓:  社交媒体 API (成本高、受限制)
```

## 维度 2: 内容处理 Pipeline

| 阶段 | 可选方案 | 复杂度 | 价值 | 必选? |
|------|---------|--------|------|-------|
| **采集** | RSS/API/抓取 | ★☆☆ | ★★★ | ✅ |
| **去重** | URL 去重 → 模糊标题去重 → 语义去重 | ★★☆ | ★★★ | ✅ |
| **分类** | 规则关键词 → LLM 分类 → ML 分类 | ★★☆→★★★ | ★★☆ | ✅ |
| **评分** | 规则评分 → AI 评分 (LLM) → ML 模型评分 | ★☆☆→★★★ | ★★★ | ⚠️ 推荐 |
| **摘要** | 截取片段 → LLM 摘要 → 多角度摘要 | ★☆☆→★★☆ | ★★★ | ⚠️ 推荐 |
| **翻译** | 无 → LLM 翻译 → 专业翻译 API | ★☆☆ | ★★★ | ⚠️ 中文用户必选 |
| **富化** | 无 → 背景搜索 → 社区评论收集 | ★★☆→★★★ | ★★☆ | 可选 |
| **聚类** | 无 → TF-IDF 聚类 → LLM Storyline | ★★★ | ★★★ | ⚠️ 推荐 |

### 处理深度选择

```
简单模式 (MVP):  采集 → 去重 → 分类 → 展示
标准模式 (v1.0): 采集 → 去重 → 评分 → 摘要 → 翻译 → 展示
高级模式 (v2.0): 采集 → 去重 → ML评分 → 聚类 → 摘要 → 翻译 → 富化 → 展示
```

## 维度 3: 个性化与推荐

| 个性化层级 | 技术方案 | 用户体验 | 实现难度 |
|-----------|---------|---------|---------|
| **L0 - 统一推送** | 相同的摘要给所有人 | ❌ 无个性 | ★☆☆ |
| **L1 - 关键词过滤** | 基于关键词的包含/排除 | 😐 基础 | ★☆☆ |
| **L2 - 源选择** | 用户选择信源和权重 | 🙂 可控 | ★☆☆ |
| **L3 - 兴趣画像** | 用户主动填写兴趣领域 | 😊 精准 | ★★☆ |
| **L4 - 隐式学习** | 从点击/阅读/星标行为学习 | 😄 自适应 | ★★★ |
| **L5 - 知识图谱** | 构建用户技术栈知识图谱 | 🤩 深度 | ★★★★ |
| **L6 - 主动发现** | 基于 gap 分析推荐未接触领域 | 🚀 超越认知 | ★★★★★ |

### 推荐路径

```
MVP:   L0 + L2 (统一推送 + 源选择)
v1.0:  L1 + L3 (关键词 + 兴趣画像)
v2.0:  L1 + L4 (关键词 + 隐式学习)
v3.0:  L5 + L6 (知识图谱 + 主动发现)
```

## 维度 4: 推送与展示

| 渠道 | 优点 | 缺点 | 适合场景 |
|------|------|------|---------|
| **Web 仪表盘** | 交互丰富、可视化 | 需要主动访问 | 深度阅读 |
| **邮件日报** | 推送被动接收、格式规范 | 易被忽略 | 每日固定时间 |
| **微信/飞书/钉钉** | 强触达、国内普及 | 需申请 Bot | 重要警报 |
| **GitHub Pages** | 免费托管、静态存档 | 无推送能力 | 历史存档 |
| **Obsidian/Notion** | 个人知识库集成 | 配置复杂 | 知识管理 |
| **MCP Server** | Agent 可调用 | 需要 AI 客户端 | 高级用户 |

### 推荐组合

```
OhMyInfo 推送策略:
  ┌─ 核心: Web 仪表盘 (技术雷达 + 日报)
  ├─ 默认: 邮件日报 (每日摘要)
  ├─ 增强: 微信推送 (重要消息)
  └─ 存档: GitHub Pages (历史记录)
```

## 维度 5: 部署模式

| 模式 | 成本 | 维护 | 隐私 | 扩展性 | 代表项目 |
|------|------|------|------|--------|---------|
| **GitHub Actions + Pages** | $0 | 低 | 中 | 中 | Horizon, TechSentry |
| **Docker 自托管** | $5-20/月 | 中 | 高 | 高 | Signal, News Diet |
| **Vercel/Railway** | $0-20/月 | 低 | 低 | 高 | Sipply, StackPulse |
| **本地运行 (CLI)** | $0 | 低 | 最高 | 低 | feed-curator, Devloop |
| **桌面应用** | $0 | 中 | 高 | 中 | ProjectBridge |

### 推荐路径

```
MVP:      GitHub Actions + Pages ($0)
Growth:   Docker 自托管 ($5-10/月)
Scale:    Vercel + Railway ($免费额度足够)
```

## 维度 6: AI 模型选择

| 模型 | 评分质量 | 摘要质量 | 成本 | 延迟 | 隐私 |
|------|---------|---------|------|------|------|
| **GPT-4o-mini** | ★★★★ | ★★★★ | $0.15/百万 token | 快 | ❌ |
| **Claude Haiku** | ★★★★ | ★★★★★ | $0.25/百万 token | 快 | ❌ |
| **Gemini Flash** | ★★★ | ★★★★ | 免费额度充裕 | 快 | ❌ |
| **DeepSeek** | ★★★★ | ★★★★ | 极低 | 中 | ⚠️ |
| **本地 Ollama** | ★★ | ★★★ | $0 | 慢 | ✅ 完全隐私 |
| **规则/ML 混合** | ★★★ | N/A | $0 | 最快 | ✅ |

### 推荐策略

```
评分:   规则 + ML 模型 (免费、快速、可离线)
摘要:   GPT-4o-mini / Gemini Flash (低成本高质)
翻译:   与摘要共用 LLM (减少额外调用)
本地:   Ollama 作为 Fallback (零成本模式)
```

## 维度 7: 变现与商业模式

| 模式 | 描述 | 可行性 | 代表 |
|------|------|--------|------|
| **开源免费** | MIT 协议, 社区驱动 | ✅ | 大部分项目 |
| **SaaS 托管** | 收费托管服务 | ✅ | auto-news (dotsfy.com) |
| **企业版** | 团队协作功能 | ✅ | TrendRadar |
| **捐赠/赞助** | OpenCollective/GitHub Sponsor | ⚠️ | 部分项目 |
| **增值 API** | 提供数据 API 收费 | ⚠️ | DataCube |
| **内容变现** | 付费订阅高级日报 | ⚠️ | 少 |

### 建议 (OhMyInfo)

```
初期:    纯开源免费 (积累用户)
中期:    开放 API + 高级特性付费
长期:    企业团队版 (Team Radar + 协作)
```
