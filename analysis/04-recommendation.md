# OhMyInfo — 推荐方案与产品定位

> 基于行业全景分析的最佳实践推荐
> 为 OhMyInfo 选择最适合的起点和发展路径

---

## 一、核心定位

```
OhMyInfo = 技术信息差消除助手
           │
           ├── 自动聚合 → 打破信息碎片化
           ├── AI 处理  → 消除语言/理解壁垒
           └── 个性化推荐 → 匹配兴趣与技能水平
```

### 一句话描述

> **OhMyInfo 是一个开源的、AI 驱动的个人技术情报系统，自动收集、理解、筛选全球技术动态，以你最能消化的方式呈现，让你永远站在技术前沿。**

## 四、MVP 功能规格 (Phase 1)

### 4.1 数据源

| 源 | 类型 | 优先级 | 采集方式 |
|----|------|--------|---------|
| Hacker News | API (Firebase) | P0 | 免费, 无 Key |
| GitHub Trending | 页面解析 | P0 | 无 Key |
| Reddit (r/MachineLearning, r/programming) | JSON API | P0 | 无 Key |
| arXiv (cs.AI, cs.LG, cs.CL) | RSS | P0 | 免费 |
| Dev.to / Lobste.rs | API | P1 | 免费 |
| Product Hunt | API | P1 | 需 Token |
| 微信公众号 (精选) | RSSHub | P2 | 需配置 |

### 4.2 处理流程

```
每 6 小时触发 (GitHub Actions):

Step 1: 并行采集所有数据源 (asyncio, ~30s)
Step 2: URL 去重 + 模糊标题去重 (~2s)
Step 3: LLM 评分 (0-10, 基于新颖性/深度/相关性) (~10s/item)
Step 4: LLM 中文摘要生成 (~15s/item)
Step 5: 按类别生成日报 Markdown (~5s)
Step 6: 部署到 GitHub Pages (~30s)

总共: ~2-3 分钟/轮
```

### 4.3 输出格式

```markdown
# OhMyInfo 技术日报 — 2026-06-20

## 🔥 今日热点
1. [项目名] - LLM 评分 9.2 - [中文摘要]
2. [项目名] - LLM 评分 8.7 - [中文摘要]

## 📡 工具发布
- **ToolName** ⭐ 9.1 | 中文说明 | [原文链接]

## 📚 研究前沿
- **Paper Name** ⭐ 8.5 | 中文摘要 | [arXiv]

## 📰 行业动态
- **News Item** ⭐ 7.8 | 中文摘要 | [源链接]

## 📊 技术趋势分析
[AI 生成的当日趋势洞察, 跨源综合]
```

## 五、关键技术决策

### 5.1 为什么 Python 优先?

| 维度 | Python | TypeScript | Go/Rust |
|------|--------|-----------|---------|
| 数据采集库 | ✅ feedparser + httpx + Scrapy | ⚠️ 相对少 | ⚠️ |
| LLM 集成 | ✅ openai + langchain | ✅ Vercel AI SDK | ⚠️ |
| 数据处理 | ✅ pandas + scikit-learn | ⚠️ | ❌ |
| 部署灵活性 | ✅ GitHub Actions + Docker | ✅ Vercel | ✅ |
| 生态成熟度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **推荐** | **MVP 首选** | **Web 优先可选** | 暂不推荐 |

### 5.2 LLM 选型

| 任务 | 推荐模型 | 原因 | 成本估算 (每日) |
|------|---------|------|----------------|
| 评分 (0-10) | GPT-4o-mini | 够用 + 便宜 | ~$0.02 |
| 摘要生成 | Gemini Flash | 免费额度大 | ~$0.01 |
| 中文翻译 | 与摘要共用 | 减少调用 | $0 |
| LLM Rerank | Claude Haiku | 质量高 | ~$0.03 |
| **总计** | | | **~$0.06/天 ≈ $1.8/月** |

### 5.3 存储方案

```
MVP:     JSON 文件 (Git 提交) — 零依赖
v1.0:    SQLite — 单文件数据库
v2.0+:   PostgreSQL (Neon 免费版) — 可扩展
```

## 六、竞品差异化

| 竞品 | 弱点 | OhMyInfo 机会 |
|------|------|-------------|
| **Horizon** | 偏通用新闻, 缺乏个性化 | 专注开发者技术雷达 |
| **TechSentry** | 无 Web UI, 只有 md 文件 | 交互式仪表盘 |
| **Signal** | 无中文, 部署复杂 | 中文优先 + 一键部署 |
| **Sentinel Feed** | 数据存 Vercel Blob 不可控 | 本地存储, 完全自主 |
| **TrendRadar** | 偏舆情监控, 非技术雷达 | 聚焦开发者技术生态 |
| **Devloop** | 纯 CLI, 无聚合展示 | Web 仪表盘 + 推送 |
| **auto-news** | Notion 依赖, 不再维护 | 自包含, 无外部依赖 |
